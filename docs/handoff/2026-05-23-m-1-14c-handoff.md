## 2. 🚨 頭號決策點(新 session 必先問 Sean 拍板,動手前):zod v3 vs v4

**實況(Code 已查):**
- node_modules 實裝 **zod 4.4.3**(transitive、沒有任何 package 宣告它)。
- `packages/schemas/package.json` **完全沒宣告 zod 依賴**(只有 devDep typescript)。
- **PRD §6 的 zod code 是 zod v3 API 寫法**,與 v4.4.3 有 breaking 差異,直接照貼會壞:

| PRD §6(v3 寫法) | zod v4 差異(需用 context7 查證確切語法) |
|---|---|
| `z.string().email('msg')` | v4 `.email()` 已 deprecated、改 top-level `z.email()`;自訂訊息 API 也改 |
| `z.literal(true, { errorMap: () => ({ message }) })` | v4 移除此 `errorMap` 位置、改 `{ error: ... }` / `{ message }` |
| `.superRefine((data, ctx) => ctx.addIssue({ code: 'custom', ... }))` | v4 `superRefine` deprecated、改 `.check()`;`addIssue` 的 issue 形狀也改 |
| `z.enum([...])` / `.default()` / `.int()` / `.min()` / `.max()` | 大致相容、但 default/error 行為細節 v4 有調整 |

**必做兩件(無論哪個版本):**
1. 把 zod **宣告成 `@pcm/schemas` 的 dependency**(目前沒宣告 = pnpm 嚴格模式下 import zod 會 phantom-dep 失敗 / lint 抓)。
2. schema code 用**正確版本的 API**(用 context7 查 zod 對應版本文件,**不憑記憶**——對齊 CLAUDE.md context7 段)。

**前後邏輯重點(M-1-14c 要理解的):**
- **schema ≠ domain type ≠ DB row**。schema 是「表單 input 子集 + 驗證」;domain type(M-1-14b `Customer`/`CustomerAddress`…)多了 id/customerUserId/createdAt 等 server 生成欄;DB row 是 snake_case。三者由 use-case + adapter mapper 串接。M-1-14c 只做 **input 驗證層**。
- **查詢(讀)路徑不走 input schema**:listAddresses / getCurrentCustomer 等讀操作直接回 domain type、不經 input schema。schema 主要服務**寫/異動路徑**(register/login/CRUD/deposit)。
- **server 端重新驗證**:同一份 schema,use-case(server)收到 client 送的資料要 `.parse()` 再驗一次(client 驗證可繞過)。tier / wallet_balance 等敏感欄**不在任何 input schema**(client 寫不到、對齊 M-1-14a column GRANT + RLS)。
- **schema 驗證須對齊 DB CHECK(M-1-14a)**:雙層防線——
  - AddressInput company → title + taxId 必填;donate → donateCode 必填(對齊 DB `addresses_invoice_company_has_data` / `addresses_invoice_donate_has_code` CHECK)。
  - DepositInput amount 正整數(對齊 DB `wallet_amount_sign` CHECK + integer 欄)。

## 4. design 細節(grep design 真權威字面、逐 schema)

> 來源:`design-reference/components/AccountPages.jsx`(637dafc)+ `WalletTab.jsx`。新 session 落地時建議再 grep 一次確認。

### LoginInput(AccountPages L181-227)
- 欄位:`email`(type=email、placeholder `your@email.com`)/ `password`(type=password、placeholder `至少 8 碼`)/ `remember`(checkbox、design default **true**)。
- design 驗證:`if (!email || !password) → 「請輸入 Email 與密碼」`。
- PRD §6:email 格式、password min 8、remember default true。

### RegisterInput(AccountPages L256-299)
- 欄位(design 順序):`name`(`王小明`、required)/ `email`(`your@email.com`)/ `phone`(`0912 345 678`)/ `password`(`至少 8 碼`)/ `agree`(checkbox)。
- design 驗證:`if (!name||!email||!password) → 「請填寫必要欄位」;if (!agree) → 「請同意服務條款」`。
- PRD §6:name min 1、email 格式、password min 8、phone regex `/^[\d\s-]{8,}$/`、agree literal true。
- ⚠️ design 欄位順序 name→email→phone→password,PRD §6 是 name→email→password→phone。schema 欄位順序純宣告、不影響 UI,但建議對齊 design 順序(鐵則 1)。

### AddressInput(AccountPages InlineAddressForm L686-757)
- 欄位:`isDefault`(checkbox「設為預設地址」)/ `name`(收件人、required、`王小明`)/ `phone`(`0912 345 678`、選填)/ `line`(地址、required、`縣市 / 區 / 路 / 號 / 樓`)。
- `invoice.type` 三 tab:`personal`=個人 / `company`=公司(三聯式)/ `donate`=捐贈。
  - personal:`carrier` 手機載具(**選填**)、placeholder `/ABCD123`。
  - company:`title` 公司抬頭(`例:賓士機車有限公司`)+ `taxId` 統一編號(placeholder `8 碼數字`、**maxLength 8**)。
  - donate:`donateCode` 愛心碼(`例:8585(罕病)、925(伊甸)`)。
- design 驗證:`if (!name.trim() || !line.trim()) return`(name + line 必填)。
- PRD §6 superRefine:company → title 必填 + taxId `/^\d{8}$/`;donate → donateCode 必填。對齊 DB CHECK。

### VehicleInput(AccountPages InlineVehicleForm L760-798)
- 欄位:`isPrimary`(checkbox「設為主要車輛」)/ `name`(車型、required、`YAMAHA YZF-R6`)/ `year`(`2022`)/ `engine`(`RJ27-xxxxx`)/ `km`(`12,340 km`)/ `mods`(`7 件`)/ `service`(type=date、最近保養)。
- design 驗證:`if (!name.trim()) return`(僅 name 必填)。
- PRD §6:isPrimary default false、name min 1、其餘 default ''、service 為 ISO date 字串 or ''。
- year/km/mods 全 string(design 是 text input、km 含千分位+單位)。

### ProfileInput(AccountPages profile tab L662-671)
- 可改欄位:`name`(姓名)/ `phone`(手機)/ `birthday`(type=date)。
- ⚠️ `email` 在 design 是 **disabled**(不可改)→ ProfileInput **不含 email**(對齊 PRD §6 + ICustomerRepository.update patch 限定 name/phone/birthday + DB column GRANT 只開這 3 欄 UPDATE)。
- PRD §6:name min 1、phone default ''、birthday default ''(ISO date or '')。

### DepositInput(WalletTab L150-224)
- design:presets `[3000, 10000, 30000, 50000, 100000]`(UI 快捷鍵、**非 schema 欄位**)、`amount`(type=number、min 100、step 100、default 10000、`disabled` when < 100)、`paymentMethod` radio:`tappay`=信用卡(TapPay)/ `atm`=ATM 轉帳(default `tappay`)。
- PRD §6:amount int、min 100(`最少 NT$ 100`)、max 1_000_000(`單次上限 NT$ 1,000,000`)、paymentMethod enum `['tappay','atm']`。
- ⚠️ design 無上限、PRD 加 max 1M(業務防呆 cap、非 design 字面、合理保留)。amount 為整數(對齊金額禁浮點 + DB integer)。
