## 3. 不可妥協落地要點

- **server 端重新驗證(鐵則)**:所有 input server 端用 `@pcm/schemas`(zod、M-1-14c)**re-parse**、不信 client 送的欄位;tier 絕不從 client 取。
- **wallet 是 server-only**:SupabaseWalletAdapter 從 `@pcm/adapters/server` import(不可從 'use client' 檔 import);deposit-wallet use-case 必 server-side。
- **雙 client 注入**:wallet adapter 建構需 readClient(帶 user session 的 authenticated client、**非裸 anon**)+ writeClient(service_role、`createSupabaseServiceClient`、server-only)。client 來源正確性由本 e slice wire-up 負責(codex 關卡1 d-2 finding)。
- **addEntry 授權在 use-case**:adapter 不做 authorization;deposit-wallet 須先驗證 current user 後才呼叫(service_role 可寫任意 customer_user_id)。
- **金額**:amount signed int(deposit +)、禁浮點;餘額靠 trigger、use-case 不自算。
- **register 走 Auth SDK 非 adapter**:customers row 由 DB trigger 自動建、use-case 不顯式 insert(對齊 ICustomerRepository JSDoc 邊界 + PRD Q2=A)。
