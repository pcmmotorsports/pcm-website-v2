## 3. 不可妥協落地要點

1. **金額鐵則(server 端)**:`amount` 是 **signed integer**(deposit 正)、**禁浮點**;整數性/sign 由 DB CHECK 守。use-case 收已驗證 domain `amount: number`(正整數驗證在 delivery 層 `DepositInput`、boundary A、use-case 不 re-parse)。

3. **ledger immutable + 餘額自動同步**:deposit = `addEntry` 一筆(`entryType:'deposit'`、`amount` 正、`relatedOrderId:null`);**不要手動改餘額**(DB trigger 同步)。**非** e-2 的「兩步 unset→set」pattern——單筆 immutable insert、無 swap、無遞補。
4. **boundary A**:deps 只 `@pcm/domain` + `@pcm/ports`(無 schemas、lint boundaries 須綠);簽名收窄(`Omit` 掉 id/createdAt;`customerUserId` 由 use-case 用 `currentUserId` 填、不信 input;`entryType:'deposit'` 由 use-case 設、不由 caller 傳任意 type)。
5. **信任邊界**:`currentUserId` server session(caller 傳、非表單 body);deposit 只能記自己的;mock 階段無真金流但金額仍 server 驗。
6. **mock 邊界誠實**:e-3 不接 TapPay(M-3);commit 訊息/STATUS 寫明「mock 記帳、真金流 M-3」、不誇大成「儲值功能完成」。
