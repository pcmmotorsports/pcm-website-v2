## 5 硬要求設計(陪審腦查證後)

### 1. Header auth-state(client、純 cosmetic)
- 鏡像既有 **CartContext hydration 範式**(`CartContext.tsx` L9-12:initial 安全空值 → `useEffect` mount 後 hydrate、`isHydrated` flag、無 SSR mismatch)。
- Header(`'use client'`):initial `isAuthed=false`(SSR 安全預設、與 server render 一致、無 mismatch)→ `useEffect` mount 後用 browser supabase client(`@/lib/supabase/browser`)查 auth-state → setState。
- 若用 `supabase.auth.onAuthStateChange` 訂閱:cleanup **必** `data.subscription.unsubscribe()`(`react-hooks/exhaustive-deps` 開著、deps 列正確)。
- `NAV_ROUTE_MAP.account` 改條件:`isAuthed ? '/account' : '/login'`。
- 程式碼註解明寫:「Header auth-state 純 cosmetic;真守門在 /account server 端 getUser()、Header 指錯也擋得住」。

### 2. 登出走 server action
- 新 `app/account/actions.ts`(`'use server'`,鏡像 `app/login/actions.ts`):`getAuthService()` → `logoutCustomer(authService)`(use-case 已存在 `packages/use-cases/src/logout-customer.ts`、走 `IAuthService.signOut` 清 session cookie)→ `redirect('/login')`。
- AccountView(client)登出 button 呼叫此 server action,**不在 client 直接 supabase.signOut**。
- 查證:storefront 現零登出路徑;logoutCustomer use-case 已備、無 delivery 接點。

### 3. acc-head name 來源(明寫)
- **採 `user_metadata.name`(輕量、免 DB)**:email 註冊(f1-b register action `metadata:{name,phone}`)+ LINE(`line-admin.ts:71` `user_metadata:{name,line_email}`)**皆寫 `user_metadata.name`**。
- acc-head:`name = user.user_metadata?.name ?? ''`(空則 avatar 退 'P' 首字、name 區顯 email);`email = user.email`;avatar = name 首字大寫。
- **g-1 head 不讀 customers 表**(自讀 RLS 留 g-4 profile 真接時)。

### 4. AccountView = 薄 router 架構
- `AccountView`(client):`tab` state + acc-head + acc-nav + 依 tab 渲染對應 tab component(薄 switch / registry)。
- 7 tab body 各自獨立檔 `components/account/tabs/{Overview,Orders,Wallet,Favorites,Vehicles,Address,Profile}Tab.tsx`;g-1 = **最小 stub**(section header 文案字面 + 「（本段於 g-X 接入）」placeholder、無 mock、無 state)。
- 目的:g-5/g-6(address/vehicle 接真資料、最肥)各自填自己的 tab 檔、不撐爆 AccountView(鐵則 6 ≤400)。

### 5. /account server 守門(getUser、force-dynamic)
- `app/account/page.tsx`(Server Component、`export const dynamic = 'force-dynamic'`):
  - `const supabase = await createServerSupabaseClient()`(`@/lib/supabase/server`、storefront 自有 lib、**不違 boundary**;Server Component 唯讀 cookie、setAll try/catch 無害)。
  - `const { data: { user } } = await supabase.auth.getUser()`(**getUser 向 auth server 驗 JWT、非可偽造的 getSession**)。
  - `if (!user) redirect('/login')`(`next/navigation`)。
  - 取 user → `<AccountView user={{ name, email }} />`。
- **直打網址照樣擋**(真安全閘);Header 路由(#1)純裝飾、指錯也擋得住。
