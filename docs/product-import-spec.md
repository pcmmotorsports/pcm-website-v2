# Product Import Spec — PCM 商品上架規範

> 版本:v1.0(2026-05-04 Sean 拍板定案)
> 適用:Phase 1 一次性 CSV 匯入腳本 + Phase 2 批次上傳後台(同一格式)

## 1. 上架方式

- **Phase 1**:Sean 手動整理好 CSV → Claude Code 執行一次性匯入腳本批次寫入 Supabase
- **Phase 2**:在此規格基礎上補做後台批次上傳 UI(不改格式,只加介面)
- 第一波展示:先上部分商品作展示,其餘等 Phase 2 批次工具完成再補

## 2. CSV 欄位定義(v4 定案)

| 欄位 | 必填 | 說明 |
|---|---|---|
| 商品名稱 | ✅ | 第一行填,規格行留空 |
| 主商品貨號 | ✅ | 同商品所有規格行填同一個,系統靠此辨識同一商品 |
| 商品描述 | ✅ | 第一行填,規格行留空 |
| 零件廠牌 | ✅ | Akrapovic / Brembo… |
| 零件分類 | ✅ | 排氣管 / 煞車 / 碳纖維… |
| 適用車種 | ✅ | 多車種用 Alt+Enter 換行垂直列;單車種直接填 |
| 適用年份 | ✅ | 對應車種同樣換行;格式:2021-2024 或 2021-至今 |
| 規格識別碼 | 選填 | 同商品所有規格行填同一數字;無規格商品留空 |
| 規格名稱1 | 選填 | 例:顏色 / 紋路 / 尺寸 |
| 規格選項1 | 選填 | 例:消光斜紋 |
| 規格圖片1 | 選填 | 多張用分號隔開:url1;url2;url3(選此規格時切換顯示) |
| 規格名稱2 | 選填 | 有第二層規格才填 |
| 規格選項2 | 選填 | 例:左側 / 右側 |
| 零售價 | ✅ | 台幣整數 |
| 經銷價 | 選填 | 空白=不開放;store/premium_store 會員可見 |
| 庫存 | ✅ | 數字;0=缺貨。前台只顯示「有貨/缺貨」,不顯示數字 |
| 商品貨號 | 選填 | 此規格唯一 SKU 編號 |
| 商品圖片 | 選填 | 多張用分號隔開:url1;url2;url3;第一行填,規格行留空 |
| PDF規格書 | 選填 | 安裝說明 / 規格表網址 |

## 3. 多規格填法

同一商品多規格 = 多行,靠「主商品貨號」綁定:

- 第一行:填齊所有欄位
- 後續規格行:商品名稱 / 描述 / 車種 / 年份 / 商品圖片 留空;只填規格選項、規格圖片、價格、庫存、商品貨號

## 4. 適用車種多車種填法

適用車種 和 適用年份 欄用 Alt+Enter 換行垂直列,車種與年份逐行對應:

```
適用車種          適用年份
Aprilia RSV4      2021-2024
Aprilia RS 660    2021-2024
Triumph Daytona   2006-2010
BMW S1000RR       2021-2026
```

單一車種直接填一行,無需換行。

## 5. 圖片規格

- 商品圖片 / 規格圖片:填完整 https:// 網址,廠商官網圖片直接貼
- 多張圖片用分號隔開,不加空格:`url1;url2;url3`
- 沒有規格專屬圖片的規格行:規格圖片1 留空,自動顯示商品主圖

## 6. 規格照片切換行為(前台)

- 客人選規格(例:消光斜紋)→ gallery 切換到該規格的照片組
- 可左右滑動瀏覽所有規格照片
- 滑完後回到商品主圖組
- 實作 trigger:M-1 商品頁 slice

## 7. 庫存顯示原則

- 後台:填入實際庫存數字,用於庫存管理
- 前台:只顯示「有貨」/「缺貨(售完)」,不顯示數字
- 實作 trigger:M-1 商品頁 slice

## 8. Filter 車種清單

- 前台 Filter 的車種清單自動從商品 fitment 資料抓取,不需手動維護
- 有上架的車種才會出現在 Filter;未上架的車種不顯示
- 資料品質決定 Filter 品質:適用車種 / 年份填得準,Filter 就準

## 9. Phase 1 匯入腳本規格(待實作)

- 輸入:符合本規範的 CSV 檔案
- 行為:
  1. 讀 CSV,以「主商品貨號」groupBy 商品
  2. 解析「適用車種」/「適用年份」換行文字 → fitments[]
  3. 解析「商品圖片」/「規格圖片1」分號 → images[]
  4. 寫入 Supabase(對齊 `docs/architecture/supabase-schema-design.md` §2-§5)
  5. 重複執行為 upsert(以主商品貨號 + 商品貨號為 key)
- 實作 trigger:M-1-03 主實作完成後獨立 slice(對齊 backlog #90)
