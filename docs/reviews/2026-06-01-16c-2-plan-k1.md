## 3. 改動清單(file-level)

### 3.1 ports:加 findByHandle
- `packages/ports/src/IProductRepository.ts`:介面加 `findByHandle(handle: string): Promise<Product | null>;`(JSDoc:仿 findById、handle 為 SEO slug、找不到回 null)。
- `packages/ports/src/IProductRepository.contract.ts`:加 contract test 案例(save 帶 variants 的 product → findByHandle(handle) 回含 variants 的 product;不存在 handle → null)。兩實作共跑。

### 3.2 SupabaseProductAdapter:findByHandle + 變體 embed
- 新增 `PRODUCT_SELECT_DETAIL_WITH_VARIANTS` = `PRODUCT_SELECT_DETAIL` + `, product_variants_public(id, sku, spec, price_general, availability, images, sort_order)`。
  - **只給 findById + findByHandle 用**(detail 單筆查詢);list 路徑(listByCategory/Brand/Fitment/searchByKeyword)維持 `PRODUCT_SELECT_DETAIL`(不帶變體、避 N+1 jsonb 膨脹)。
- `findByHandle(handle)`:仿 findById、`.from('products_public').select(PRODUCT_SELECT_DETAIL_WITH_VARIANTS).eq('handle', handle).single()` → findSingle → mapSupabaseProductToDomain。
- findById 改用 `PRODUCT_SELECT_DETAIL_WITH_VARIANTS`(#203 規定 findById 也帶變體;findById 目前無前台呼叫點、無 N+1 風險)。
- **embed 實測確認**:PostgREST 偵測 view↔view 關係成功、price_store/metadata 因 view 排除無法 select(DB 層防護)。

### 3.3 mappers:mapVariantRow + 填 variants
- `packages/adapters/src/supabase/mappers/product.ts`:
  - `SupabaseProductRow` 加 optional `product_variants_public?: SupabaseVariantRow[]`(wire 型別、list 路徑無此 key → undefined)。
  - 新 `SupabaseVariantRow` 型別:{id, sku, spec:Record<string,string>, price_general:number|null, availability, images:string[], sort_order:number}(**不含 price_store/metadata**、對齊 view 10 欄投射)。
  - 新 `mapVariantRow(row): ProductVariant`:price_general→priceByTier.general(toMoneyAmount guard、null 則 throw 同 product);store=dummy{0,TWD};premiumStore=placeholder{0,TWD}(鏡像 mapSupabaseProductToDomain、註解同源);images 直送 string[](16b 已抽 url、shape 對齊);spec 直送 Record<string,string>;sortOrder←sort_order。
  - `mapSupabaseProductToDomain`:`variants: (row.product_variants_public ?? []).map(mapVariantRow)`(list 路徑 undefined→[]、detail 路徑填真;可選依 sort_order 排序、DB 已 ORDER 則直送)。

### 3.4 domain:computeEffectiveVariantPrice(防 NT$0)
- `packages/domain/src/catalog/pricing.ts`:加 `computeEffectiveVariantPrice(variant: ProductVariant, brand: Brand, tier: MemberTier): Money`。
  - general→variant.priceByTier.general;store→variant.priceByTier.store;premiumStore→round(store×(1-brand.premium_extra_pct/100))。
  - 🔴 **NT$0 guard(reminder 2)**:因 public view 無經銷價、變體 store/premiumStore=dummy 0。本函式語意保持與 computeEffectivePrice 對稱(收 brand 補變體不帶 brand 的缺口),但**16c-3 strip 時對變體 UI 價一律取 general**(無真經銷價可顯、防 NT$0、對齊 featured g-2 'general' 釘法)。本片只交付 domain 函式 + 在 JSDoc 標「store/premiumStore 在 public-view 路徑為 dummy 0、caller 顯示前須防 NT$0、真經銷價待 M-2-08 server-side pricing endpoint」。
- `packages/domain/src/index.ts`:export computeEffectiveVariantPrice。
