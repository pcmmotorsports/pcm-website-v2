## 1. 目標
詳情頁 ProductTabs 規格表「產品型號」由 `PCM-{id hash}` 亂碼數 → 顯真主碼(external_id、如 `RPM-DCC01`);「產地」由硬寫「義大利」→「泰國」。需把 wire `external_id` 一路接到 UI(domain Product 目前無此欄、mapper 沒搬)。

## 2. 命名決策(執行端定、Sean 委派)
domain 新欄命名 = **`externalId`**(camelCase 直鏡 wire `external_id`、provenance 清楚、JSDoc 標「← wire external_id、vendor 主碼如 RPM-DCC01」)。不取 `code`(語意較含糊「什麼的 code」)。UI 標籤維持「產品型號」(presentation ≠ domain 欄名、不需一致)。

## 3. 現況(實查)

## 9. codex 關卡1 結果(round1)+ 自修(2026-06-01)

codex round1 = **FAIL**、4 must-fix + 2 consider + security pass-point。全採納、**不跑 round2**(fixes 清楚)。修正:

- **[must-fix 1] 命名 externalId → `productCode`**:codex「externalId 直鏡 wire 違 ADR-0003 domain 業務語意」;code 太泛、sku 撞 ProductVariant.sku → **`productCode`**(商品主碼、業務語意 camelCase)。§2/§4 全 `externalId` 字面改 `productCode`;JSDoc 標「vendor 主碼 ← wire external_id」。
- **[must-fix 2] ripple + mapper read 測**:`productCode: row.external_id`(read)、`external_id: domain.productCode`(write);加 `mappers/product.test.ts` 斷 mapped.productCode === row.external_id(別只型別擋)。建構點 §4.1 已列全(mapper return + createFakeProduct + pricing.test fixture)。
- **[must-fix 3] UI fallback 命名**:ProductTabs `{product.productCode ?? product.slug}`;toUIProduct 真資料填 required productCode(不落 fallback);MockProduct.productCode? optional。
- **[must-fix 4] D2 動手前拍**:產地泰國 vs L116「義大利原廠保固」同屏矛盾、**D2 移到動手前問 Sean**(codex 薦 B「原廠保固 24 個月」最保守、不引未證實「泰國原廠保固」)。
- **[consider 5] write-path 測**:加 mapper write 測斷 external_id === domain.productCode(防 placeholder 回歸)。
- **[consider 6] 產地 hardcode 債明寫**:ProductTabs 產地「泰國」= Phase-1 RPM-only 臨時覆蓋、非通用真值;未來非 RPM 會錯、product_specs 接線前不可當通用真值。ProductTabs 加註 + backlog #162 記。
- **[security ✓]** external_id 公開料號、products_public 本就投射、不碰 pricing → 確認**不需 codex 關卡2**。

修正後 domain 欄名 = `productCode`(非 externalId);其餘 §4 結構不變。**待 Sean:plan 批准 + D1(選變體換圖排序)+ D2(L116 義大利保固)。**
