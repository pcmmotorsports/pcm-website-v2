# PCM GEO 體檢報告 — 新站(pcm-website-v2)+ 舊官網(pcmmotorsports.com)

## 1. 一句話總結

**新站的「內容血肉」與商品結構化資料底子相當好(AI 看得到、看得懂),但缺了三塊「站台骨架」(robots.txt、sitemap、正式網域變數)——這三項是上線前必補的 blocker,補完就達到 GEO 及格線;舊官網現在爬得到但「PCM 是誰、在哪、賣什麼」幾乎沒餵給 AI。**

---

## 2. 白話總結(老闆 30 秒掃讀)

GEO = 讓 Claude / ChatGPT / Perplexity 這些 AI 引擎能爬到你的網站、看懂內容、在回答客人問題時引用你。以下是現況:

1. **新站「內容看得到」這關過了(重要)** — 不跑程式的 AI 爬蟲,在新站首頁、商品列表、商品詳情頁都能讀到真實的標題、文案、價格、適用車款。很多網站敗在「內容要等瀏覽器跑程式才出現、AI 看到空白」,我們沒這問題。

2. **新站「商品結構化資料」是強項** — 每個商品頁都自動附了一張 AI 看得懂的「商品身分證」(名稱/品牌/價格/料號/常見問答),這是電商 GEO 最值錢的東西,而且經銷價零外洩、只露一般會員價。已經做好了。

3. **最大缺口是三塊「站台骨架」,上線前一定要補** — (a) `robots.txt`(招待員,跟 AI 說「歡迎爬、地圖在這」)、(b) `sitemap.xml`(地圖,把上千個商品頁一次攤給 AI,否則它只能靠連結摸黑爬、很多商品頁會漏掉)、(c) 設定**正式網域變數**(沒設的話,所有頁面的「正規網址」會整片空白,AI 抓不到你的網址)。這三項目前**完全不存在**,工很小但影響很大。

4. **現在做「機器人監控」太早** — 站還沒公開,沒有真的 AI 爬蟲會來,現在監控等於監控一間空房。應該上線後第一週再做。

5. **舊官網(現在唯一公開的站)** — 好消息:爬得到、是真實 HTML(非空殼)、有標題和社群分享標籤。壞消息:沒有公司名/地址/電話的結構化資料、沒有 robots.txt、沒有 sitemap、整頁連一個主標題(h1)都沒有。AI 被問「台灣哪裡買重機改裝零件」時,沒有結構化事實可引用。但這是過渡用的舊站,新站才是長期主場。

6. **內容層的軟肋(不擋上線,但會限制引用品質)** — 新站商品介紹目前是「碳纖維通用範本」,兩支不同商品的介紹文字一模一樣;而且 AI 最愛引用的「商品描述」欄位只塞了 12 字副標。這不是程式問題,是後台內容資料線的工作,可上線後分批補。

## 3. 上線前必補清單(blocker 優先序)

> 只列**真有 GEO 影響**的項目。「優先序」欄 P0 = 不補上線就半殘的 blocker。

| 優先序 | 項目 | 現況 | 為什麼重要(白話) | 工作量 | Blocker? |
|---|---|---|---|---|---|
| **P0** | **設定 `NEXT_PUBLIC_SITE_URL` 正式網域** | 未設(程式正確,只缺一個 Vercel 環境變數) | 沒設的話,**整個正式站**的正規網址、社群分享網址、商品身分證裡的網址**全部變空白**。AI 抓不到「這頁的官方網址是什麼」,後面補的 sitemap 也產不出正確絕對網址。一個設定救三項。 | 極小(Vercel 後台填一格) | **是** |
| **P0** | **`robots.txt`** | 不存在(連 `public/` 目錄都沒有) | AI 訓練爬蟲(ClaudeBot/GPTBot)第一步就抓這個檔。證據顯示文章中 ClaudeBot 188 次抓取幾乎全集中在 `/robots.txt` 與 `/sitemap.xml`——這兩個檔就是 AI 的「大門」。缺席等於讓 AI 摸黑爬、降低爬取意願,也無法告訴它地圖在哪。 | 極小 | **是** |
| **P0** | **`sitemap.xml`(動態 `app/sitemap.ts`)** | 不存在 | 把上千個商品頁的網址一次攤給 AI。沒有它,AI 只能靠頁面連結一層層摸,深層商品頁很可能整批漏抓,能被引用的商品大幅縮水。技術上可重用現有撈商品的程式自動產出。 | 小 | **是** |
| **P1** | **Organization / Store JSON-LD(商家身分證)** | 完全沒有 | 目前只有「商品」身分證,沒有「公司」身分證。AI 被問「PCM 是誰、賣什麼、怎麼聯絡、LINE」時無結構化來源可引。對你這種強品牌、強聯絡導向的台灣重機零件商,這是 AI 抓商家身分的關鍵槓桿(公司名 + LINE/社群 + 聯絡方式)。 | 小 | 強烈建議 |
| **P2** | **確認 Vercel「部署保護」沒開** | 需 Sean 在後台確認(repo 看不到) | 若部署啟用了平台層密碼保護(Deployment Protection),會**整站擋住所有爬蟲**,前面做的全白費。上線時務必確認關閉。 | 極小(後台確認) | 是(上線當下) |

**好消息(已到位、不用做)**:新站三大頁面內容都進初始 HTML(SSR 正常)、商品 Product JSON-LD + FAQPage JSON-LD 已上線且經銷價零外洩、各頁有專屬 title/description、`<html lang="zh-Hant">` 正確、無 middleware/header 層級的爬蟲封鎖、公開頁不需登入。這些是稽核確認的綠燈項。

## 5. 舊官網現況(pcmmotorsports.com — 現役公開站)

誠實評估,**所有判讀都是 `curl` 直抓首頁 HTML 親驗**(非臆測)。值得一提:WebFetch 工具的 markdown 轉換**誤報**了「無 title / 無 OpenGraph / 無 JSON-LD」,以 curl 親讀 head 為準——這三項其實都存在。

**驗證到「沒問題」的(綠燈)**:

| 項目 | 狀態 | 證據 |
|---|---|---|
| 首頁可達、非機器人攔截 | OK | HTTP 200、113KB 真實靜態 HTML |
| 內容在靜態 HTML(不跑程式也看得到) | OK | 約 1349 字可見中文文案 + 46 個分類連結 |
| 語言標註 / 標題 / OpenGraph | OK | `lang="zh-TW"`、`<title>PCM 重機零件販售…</title>`、完整 og 標籤 |
| 基礎站台 JSON-LD | 部分 | 有 WebPage + WebSite + BreadcrumbList(但缺商家事實層) |

**驗證到「有缺口」的(紅/黃燈)**:

| 項目 | 狀態 | 影響 |
|---|---|---|
| **Organization/LocalBusiness(公司名/地址/電話)** | **缺** | **舊站最大單點損失**——AI 被問「PCM 在哪/台灣哪買重機改裝零件」時無結構化事實可引,即使首頁掛了合作店家地圖 |
| Product/Offer 結構化資料 | 缺 | AI 無法抽「PCM 賣哪些品牌/品類/價格」 |
| robots.txt | 缺(404) | 無法宣告地圖、無法明確允許 AI 爬蟲 |
| sitemap.xml | 缺(404) | 靜態鏡射沒保留動態 sitemap,深層頁靠連結盲爬、爬不全 |
| meta description | 缺 | 部分爬蟲讀不到頁面摘要 |
| h1 主標題 | 壞 | 整頁 h1=0、h2=0,只有 h3/h4,語意大綱斷裂 |
| canonical | 壞(影響小) | 寫成相對路徑 `index.html`(靜態化副作用),應為絕對網址 |

**結論**:舊站「爬得到、語言看得懂」,但「PCM 是誰、賣什麼、在哪、怎麼聯絡」這些 GEO 引用所需的事實層幾乎沒結構化。**這是過渡用的 stopgap 站,新站才是長期 GEO 主場**——舊站修補建議只做低風險加值(補一段 Organization JSON-LD + robots + sitemap),不必大改。

## 6. 機器人監控 — 為何「照搬 FB 教學」行不通,以及正確做法

**FB 文章教的 `grep /var/log/nginx/access.log` 在我們這完全不適用,而且不是「改個路徑就能用」**:

- 我們部署在 **Vercel(serverless)**,不是自己租的伺服器。**沒有一台你能登入(SSH)的主機、沒有 nginx、沒有那個 log 檔**。每個請求由臨時容器處理、跑完就銷毀。
- Vercel 自家的即時 log **保留時間極短**(免費方案約 1 小時量級),而 AI 爬蟲可能幾天才來一次,等你想查時 log 早被沖掉,**無法累積長期觀察**。

**Vercel 上抓 AI 爬蟲的正確做法(三選一)**:

| 方案 | 適不適合我們 | 成本 |
|---|---|---|
| **(推薦)程式內偵測 bot UA → 寫進 Supabase** | 最務實。所有零件都已備齊(現成 Supabase client + serverless route 範本),之後直接 SQL 查 | 約半天~一天 |
| Vercel Log Drains 導出到外部日誌平台 | 零程式改動但**需升級 Vercel Pro 付費方案** + 接外部服務,對「只想知道 AI 有沒有來」略嫌重 | 設定 1-2 小時 + 付費 |
| Vercel Web Analytics | **錯的工具**——它靠瀏覽器跑程式回報,而 AI 訓練爬蟲不跑程式,根本不會被計到,且它設計上本來就過濾 bot | 不建議 |

**建議**:**新站公開前不急著做監控**。站還沒上線 = 沒有真爬蟲會來 = 現在監控是監控空房間,投報率為 0。監控的價值在「驗證爬蟲確實有來」,本質是上線後的後置動作。上線前該優先補的是讓爬蟲「爬得到 + 爬得對」(robots + sitemap + 網域變數);**上線後第一週**再用推薦方案輕量驗證 ClaudeBot/GPTBot 是否真的有來。

## 7. 一個比喻看懂全局

把 GEO 想成「**讓 AI 當你的店員、主動跟客人推薦 PCM**」:

- **新站的內容與商品身分證** = 你的**商品陳列做得很好**,每件貨都掛了清楚的標籤(名稱/價格/規格)。AI 走進店裡看得一清二楚。✅
- **缺 robots.txt + sitemap** = 你的店**沒有大門招牌、沒有店內地圖**。AI 想進來推薦你,卻找不到入口、也不知道你有上千件貨——只能在門口摸黑、看到幾件就走。**這是上線前一定要掛上的。**
- **缺正式網域變數** = 你的貨都很好,但**價目牌上沒寫店名地址**,AI 想告訴客人「去這家買」卻講不出你的門牌。**一個設定就補上。**
- **缺商家身分證(Organization)** = 客人問「這家店是誰、在哪、怎麼聯絡」,AI 翻遍店裡找不到你的名片。**補一張就好。**
- **現在做監控** = 店都還沒開門,就裝了一台計算「今天幾個客人上門」的機器——**先開門(上線)再裝才有意義。**

一句話:**陳列(內容)做得好,差的是大門、地圖、門牌——三樣小工掛上去,AI 才能把客人帶進來。**
